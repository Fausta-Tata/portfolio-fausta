Project Information

  • Title: Compliance Monitoring Analysis
  • Category: Excel
  • Key Tasks: Data Cleaning, Pivoting, Create Dashboard
  • Project URL: github

Project Objectives

The purpose of this analysis is to evaluate the level of compliance of the order fulfillment process with company standards (SLA), identify areas of inefficiency, and find the root cause of data quality issues to provide recommendations for improvement.

Findings

Data Quality is Still Poor

Many issues were found in the raw data, such as mixed dates and courier names in one column, missing date data, inconsistent spelling of courier names (e.g., "Sicepat" vs. "SiCepat"), and illogical data (shipping date earlier than payment date or shipping date in the future that doesn't make sense).

Time Compliance Level is Not Optimal

Even after the data was cleaned, a large number of orders were still found to have a "Tidak Patuh" (Non-Compliant) status, meaning the processing time from payment to delivery to the courier exceeded the established SLA of 5 days.

Low Completeness of Crucial Data

The number of orders do not have "Tgl_Diserahkan_Kurir" (Date_Delivered_by_Courier) or "Kurir" (Courier data), so their compliance status cannot be measured and is labeled "Data Tidak Lengkap" (Incomplete Data)

Reccomendation

Data Input System Improvement

Propose changes to the data input system to separate fields for Date and Courier. Also implement mandatory field validation for these crucial fields.

Operational Delay Investigation

Conduct a deeper analysis on orders with "Non-Compliant" status to find the root cause of the delay (e.g.: warehouse issues, handover process with a particular courier, etc.).

Historical Data Cleansing

Form a team to clean and validate historical shipping data.
